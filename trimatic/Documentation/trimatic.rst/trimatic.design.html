sweep(*args, profile, path, follow_surface_left=False, follow_surface_right=False, segment_angle=10, surface_segmentation_profile_based=False, surface_segmentation_path_based_direction_angle=None)[source]¶

Sweeps a profile along a path to create a part.

Parameters:
  • profile (trimatic.Sketch) – The profile to be swept.
  • path (trimatic.Curve) – The path to follow.
  • follow_surface_left (bool, optional) – When True, the orientation of the sweep profile during the sweep will follow the surface, so the orientation is preserved with respect to the surface left from the path. This is only possible for attached curves.
  • follow_surface_right (bool, optional) – When True, the orientation of the sweep profile during the sweep will follow the surface, so the orientation is preserved with respect to the surface right from the path. This is only possible for attached curves.
  • segment_angle (float, optional) – When the path makes curves, the triangulated surface that originates from the sweep can never make a perfect round. Defining this parameter, you determine the deviation between the triangulated surface and a ‘perfect’ round. The bigger the angle, the fewer segments, the rougher the approximation.
  • surface_segmentation_profile_based (bool, optional) – When True, an additional ‘longitudinal’ segmentation is done: each segment of the profile becomes a surface when it is swept around.
  • surface_segmentation_path_based_direction_angle (float, optional) – When specified, an additional ‘transversal’ segmentation is done: the surface is segmented when the direction of the path changes. You can set the value of an angle. When the direction changes more than this angle, the surface is split.

sweep_loft(start_profile, path, end_profile, intermediate_profiles=None, detail=0.5)[source]¶

With the Sweep-loft function, you can sweep multiple profiles along a path. Please take the following conditions into account to get a good result:

  1. Profiles must be closed (both planar and non-planar profiles are allowed).
