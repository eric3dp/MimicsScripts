create_path_based_pattern(*args, path, pattern_entity, distance=1.0, number_of_copies=10, is_follow_curve=True, is_follow_surface=False, is_transform_displacement=False)[source]¶

Patterns entities along the guiding path that is defined by the curve.

Parameters:
  • path (trimatic.Curve) – The curve that is used as the guiding path.
  • pattern_entity (trimatic.Part, trimatic.Sketch, trimatic.Plane) – The entity to be patterned along the guiding path.
  • distance (float | None, optional) – The distance between the pattern entities. If set to None, the distance will be computed automatically based on the number of copies. If number_of_copies is also None, then the pattern entities will be placed on the curve segment endpoints.
  • number_of_copies (int | None, optional) – The number of pattern entities to be added along the guiding path. If set to None, the number will be computed automatically based on the distance. If distance is also None, then the pattern entities will be placed on the curve segment endpoints.
  • is_follow_curve (bool, optional) – If True, the direction of the pattern entity will follow the curve direction.
  • is_follow_surface (bool, optional) – If True, and the path is an attached curve, the triangle normals are used to align the Z axis of the pattern entity.
  • is_transform_displacement (bool, optional) – If True, the displacement between the pattern entity and the path curve will be transformed such that the pattern entity will be centered on top of the path.
Returns:

One or multiple pattern results.
Return type:

trimatic.Part | (trimatic.Plane) | (trimatic.Sketch)
Raises:

ValueError, RuntimeError
Example: 
import trimatic
import os

trimatic.open_project(os.path.dirname(trimatic.get_application_path()) + r"\DemoFiles\AAA_trimmed.mxp")
part = trimatic.find_part('Centerline  1')
curve = part.find_curve('Small2')
sphere = trimatic.create_sphere_part((0, 0, 0), 2)
outputs = trimatic.create_path_based_pattern(path=curve, pattern_entity=sphere, is_follow_surface=False, number_of_copies=5, distance=None)
create_point_based_pattern(*args, entity, pattern_entity, specific_points_on_mesh, direction=None, rotation_angle=0.0, height_offset=0.0)[source]¶

Operation to place pattern entities on the desired points of a reference mesh.

Parameters:
  • entity (trimatic.Part) – A single part to be used as the marker for the points to place the pattern entities.
  • pattern_entity (trimatic.Part) – A single part to be used as the pattern entity.
  • specific_points_on_mesh (trimatic.Point | [trimatic.Point] | [(float, float, float)]) – One or a list of points to be used as the markers to place the pattern entities.
  • direction ((float, float, float), optional) – The direction vector for the X axis of the pattern entity to be aligned to. If set to None, all pattern entities are placed without a shared common direction.
  • rotation_angle (float, optional) – The rotation of the pattern entities about the vertical axis of the pattern entity.
  • height_offset (float, optional) – Offset of the pattern position from the specified point along the vertical axis.
Returns:

One part with the patterned result.
Return type:

trimatic.Part
Raises:

ValueError, RuntimeError
Example: 
import trimatic
import os

box_side = 10
box = trimatic.create_box_part(corner_point = (0,0,0), x_extent = box_side, y_extent = box_side, z_extent = box_side)
cone = trimatic.create_cone_part(bottom_radius = 1, height = 3, origin = (0,0,0), direction = (0,0,1))

points = [(5,5,10), (5,10,5)]
output = trimatic.create_point_based_pattern(entity = box, pattern_entity = cone, specific_points_on_mesh = points)
fillet(entities, radius=2.0, tolerance=0.01)[source]¶

Rounds (fillets) the sharp edges between surfaces to a desired radius.

Parameters:
  • entities (trimatic.SurfaceBorder | trimatic.SurfaceContour | [trimatic.SurfaceBorder | trimatic.SurfaceContour]) – Borders or contours to be rounded.
  • radius (float, optional) – The radius of the fillet.
  • tolerance (float, optional) – This parameter allows you to set the accuracy of the fillet. A lower tolerance will lead to a higher accuracy and smaller triangles, whereas a higher tolerance will lead to a lower accuracy and larger triangles. A value equal to or smaller than the radius divided by a factor of 200 usually provides a good result without leading to an excessive number of triangles being generated.
Returns:

The rounded surfaces that were created
Return type:

trimatic.Surface | (trimatic.Surface)
Raises:

RuntimeError, ValueError
Example: 
import trimatic

box = trimatic.create_box_part(corner_point=(0,0,0), x_extent=5, y_extent=5, z_extent=5)
c1 =  box.find_surface("Top").get_border().get_contours()[0]
c2 =  box.find_surface("Right").get_border().get_contours()[0]
trimatic.fillet(entities=(c1,c2), radius=0.3, tolerance=0.0015)
quick_label(*args, entity, text, point, direction, follow_surface=True, alignment=<TextAlignment.Center: 'Center'>, font='Arial', font_height=4, label_height=1, bold=False, italic=False)[source]¶

Places a label on a part based on text.

Parameters:
  • entity (trimatic.Part) – Entity to label.
  • text (string) – Text to use for label. Can be multiline. Use <backslash>n for new lines.
  • point (trimatic.Point) – Point where the label will start.
  • direction (trimatic.Point) – Direction of the label.
  • follow_surface (bool, optional) – When True, the label will be bent to follow the surface normal. When False, the label will be aligned with the normal at the starting point.
  • alignment (trimatic.TextAlignment, optional) – The text alignment.
  • font (string, optional) – Font to use for the label.
  • font_height (float, optional) – Font height
  • label_height (float, optional) – Label height (embossed text). Use a negative value for labeling inwards (debossed text).
  • bold (bool, optional) – Set to True for bold font style.
  • italic (bool, optional) – Set to True for italic font style.
Returns:

SurfaceSet containing surfaces that were created for the label.
Return type:

trimatic.SurfaceSet
Raises:

RuntimeError, ValueError
Example: 
import trimatic

size = 100
box = trimatic.create_box_part([0, 0, 0], size, size, size)
trimatic.quick_label(entity=box, text='front->up', point=(size / 4, size / 4, size), direction=(0, 1, 0))

remove_undercut(undercut_entity, fitting_entity, direction, draft_angle=0.0, detail=0.3)[source]¶

Removes excess undercuts from a part, such that it fits into another part according to a set direction.

Parameters:
  • undercut_entity (trimatic.Part) – Part that should have its undercuts removed.
  • fitting_entity (trimatic.Part) – Part that the undercut entity should fit into after the operation completes.
  • direction (trimatic.Point) – The direction to take out the undercut entity from the fitting entity.
  • draft_angle (float, optional) – The draft angle by which you want to take out the undercut entity from the fitting entity.
  • detail (float, optional) – Smaller input here gives a smoother cut surface at the cost of increased computation time.
Returns:

Part after removal of undercut.
Return type:

trimatic.Part
Raises:

RuntimeError, ValueError
Example: 
import trimatic

box = trimatic.create_box_part([0, 0, 0], 10, 15, 25)
cylinder = trimatic.create_cylinder_part([0, 0, 0], [0, 0, 20], 2.5)

result = trimatic.remove_undercut(undercut_entity=box,
                                  fitting_entity=cylinder,
                                  direction=[0, 0, 1],
                                  draft_angle=5.0,
                                  detail=0.3)
